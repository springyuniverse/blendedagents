## 4. Launch-Day SEO Checklist

### Technical SEO

- [ ] **Canonical URLs:** Set `<link rel="canonical">` on every page. Landing page should canonical to `https://blendedagents.com/`
- [ ] **SSL:** Verify HTTPS everywhere with no mixed content warnings
- [ ] **Site speed:** Target Core Web Vitals -- LCP <2.5s, FID <100ms, CLS <0.1. Current landing is static HTML which is excellent for this
- [ ] **Mobile responsiveness:** The landing page has responsive breakpoints at 768px. Verify all pages pass Google's mobile-friendly test
- [ ] **Sitemap:** Generate and submit `sitemap.xml` to Google Search Console. Include:
  - Landing page (`/`)
  - `/signup/builder`
  - `/signup/tester`
  - `/login`
  - `/skill.md`
  - All pillar pages once published
  - All blog posts once published
- [ ] **Robots.txt:** Allow all public pages. Block `/admin/*`, `/api/*`, `/builder/*` (authenticated dashboard), `/tester/*`
- [ ] **Google Search Console:** Verify domain ownership. Submit sitemap. Request indexing for homepage
- [ ] **Bing Webmaster Tools:** Submit there too -- Bing powers Copilot and ChatGPT web search
- [ ] **Structured data (see below)**
- [ ] **404 page:** Custom 404 with navigation back to homepage and search
- [ ] **Page speed:** Preload critical fonts (Geist, JetBrains Mono). Defer non-critical JS. Inline critical CSS (already done in landing)
- [ ] **URL structure:** Clean, lowercase, hyphenated. No trailing slashes or query parameters for content pages
- [ ] **Internal linking:** Every page should be reachable within 3 clicks from homepage

**Data source:** Curate a list of 50-100 common web app features. For each, generate:

**Keyword pattern:** "how to test [feature]" + "test [feature] with AI" -- collectively targets thousands of monthly searches.

## 7. Competitor SEO Gaps -- What to Steal

### Testlio (DR ~62, ~45K organic monthly visits)

**Keywords Testlio ranks for that we can target:**
- "crowdsourced testing" (Testlio ranks #3-5) -- We can create `/guides/crowdsourced-testing-for-ai` to flank
- "managed testing services" (Testlio ranks #2-4) -- Our pay-per-test model is the differentiator angle
- "payment testing" (Testlio owns this niche) -- We can create `/test/payment-integration` to capture adjacent traffic
- "localization testing" (strong Testlio content) -- Skip this -- not our core value prop

**Gap:** Testlio has zero content about AI agents, agentic testing, or human-as-a-tool. Their entire SEO is built around traditional managed testing. We own the AI-agent angle completely.

### Rainforest QA (DR ~55, ~20K organic monthly visits)

**Keywords Rainforest ranks for:**
- "no-code testing" (Rainforest ranks well) -- Not our angle, skip
- "Rainforest QA alternative" (~500/mo) -- Create `/compare/rainforest-qa` immediately
- "self-healing tests" (Rainforest content) -- Different approach; we complement rather than compete
- "QA automation for SaaS" (Rainforest blog ranks) -- Create content that positions human verification as the layer *on top of* automation

**Gap:** Rainforest positions as "automation + crowd." They don't address the developer building with AI coding tools at all. Their pricing ($1,500-$6,500/mo) prices out solo devs and small teams. Our `/for/startups` and `/for/indie-hackers` pages capture this abandoned segment.

### QA Wolf (DR ~50, ~15K organic monthly visits)

**Keywords QA Wolf ranks for:**
- "end to end testing service" (QA Wolf ranks #1-3) -- Create content on "e2e testing with human verification"
- "Playwright testing service" (QA Wolf owns this) -- Create `/test/playwright-apps` focusing on human verification layer
- "80% test coverage" (their core marketing claim) -- Counter-position: "100% of *critical paths* verified by humans"
- "QA Wolf alternative" (~500/mo) -- Create `/compare/qa-wolf` emphasizing no subscription, API-first

**Gap:** QA Wolf is a $3,500+/mo service with human QA engineers, not an API. They cannot serve the "I want to test one feature right now for $1.30" use case. Every piece of content we create should emphasize pay-per-test economics and API-first integration.

### Universal Gaps Across All Competitors

1. **None of them have content about AI agent tool-use.** Zero results for "human as a tool for AI agents" from any competitor.
2. **None address the "vibe coding" / AI-generated code verification problem.** This is a massive emerging search cluster.
3. **None offer skill.md or agent-native integration content.** They're all built for humans to configure, not for AI agents to call.
4. **None have comparison pages against each other.** First mover on `/compare/` pages wins those SERPs.
5. **None target the indie developer / solo builder segment.** Their pricing and positioning is enterprise-first.

## 8. "Viral SEO" Plays -- Content Designed to Earn Links and Shares

### Play 1: "The Human API" Concept Page
**URL:** `/human-api`
**Concept:** A beautiful, interactive page that shows the API spec for "calling a human." Treat it like API docs but the endpoint is a person. Visual design similar to Stripe's API docs but with a human twist. The juxtaposition is inherently shareable.
**Why it goes viral:** The framing is provocative and novel. Tech Twitter, Hacker News, and AI newsletters will pick it up because it crystallizes the Human-as-a-Tool concept in a visually memorable way.

### Play 2: "AI vs Human: Can Your AI Agent Spot These Bugs?"
**URL:** `/blog/ai-vs-human-bug-detection`
**Concept:** Take 20 real-world UI bugs (with screenshots). Show what automated testing found vs. what human testers found. Let readers guess which ones the AI missed. Interactive quiz format.
**Why it goes viral:** Developers love being proven wrong. The AI-vs-human framing generates debate. Shareable quiz format. Data-backed.

### Play 3: "The skill.md Standard" -- Open Specification
**URL:** `/skill-md`
**Concept:** Propose skill.md as an open standard for AI-agent-to-human-service communication. Publish a specification. Invite other services to adopt it. Create a GitHub repo for the spec.
**Why it goes viral:** Standards earn backlinks permanently. If even 5 services adopt skill.md, it becomes a canonical reference. Developer community loves open specs.

### Play 4: "Ship It or Fix It" -- Weekly Bug Gallery
**URL:** `/blog/ship-it-or-fix-it` (recurring series)
**Concept:** Weekly post showing anonymized real bugs found by BlendedAgents testers. Readers vote: "Would you have caught this?" Screenshots, screen recordings, severity ratings.
**Why it goes viral:** Recurring content builds audience. Visual bugs are inherently shareable. Developers relate to "I've shipped that bug before." Builds brand as the place where bugs get caught.

### Play 5: "The $1.30 QA Test" Calculator
**URL:** `/pricing` (embed interactive calculator)
**Concept:** Interactive widget where you describe your test (number of steps, devices, type) and it shows exactly what it costs. Compare to the hourly rate of a QA engineer ($40-80/hr) or the cost of a Testlio/QA Wolf subscription.
**Why it goes viral:** Price transparency is rare in QA. The contrast between $1.30 and $3,500/mo is attention-grabbing. Developers share pricing calculators.

### Play 6: "What AI Agents Can't See" -- Visual Essay
**URL:** `/blog/what-ai-agents-cant-see`
**Concept:** A beautifully designed long-form piece showing side-by-side: what an AI agent "sees" (DOM, test results, console output) vs. what a human sees (visual hierarchy, emotional response, confusion, delight). Use real examples.
**Why it goes viral:** Taps into the AI hype/skepticism debate. Visual format works on social. Makes a philosophical point with practical examples.

## 10. Answer Engine Optimization (AEO) -- Getting Featured in AI Search

### The Opportunity

ChatGPT processes queries from 700M+ weekly users. Perplexity is growing fast. Google AI Overviews reach 2B monthly users. For BlendedAgents, the goal is to be the cited source when someone asks an AI: "How do I get my AI agent to test my app?" or "What is human-in-the-loop testing?"

### Strategy: Optimize for Citation, Not Just Ranking

**The rule:** Unlike SEO, there is no "page 2" in AI search. You are either cited or invisible.

### Tactical Playbook

#### 1. Structure Content for Extraction
Every pillar page and blog post should follow this format:
- **First paragraph:** Direct, factual answer to the target question in 2-3 sentences
- **Second paragraph:** Supporting context and nuance
- **Body:** Deep detail with headers, lists, and examples
- **FAQ section:** 3-5 questions at the bottom in Q&A format

AI models extract from the first 200 words disproportionately. Front-load the answer.

#### 2. Claim the Definition
Create a dedicated page for every term we want to own:
- `/glossary/human-as-a-tool` -- Define this term before anyone else does
- `/glossary/agentic-qa` -- Our definition with our framing
- `/glossary/human-in-the-loop-testing` -- Comprehensive definition
- `/glossary/skill-md` -- Document the concept

These glossary pages should be concise (500-800 words), definitional in tone, and structured with schema markup (`DefinedTerm` schema type).

#### 3. Be the Cited Source for Comparisons
When someone asks an AI "What's the best managed QA testing service?", the AI will cite comparison pages. Our `/compare/` pages and buyer's guide pillar should:
- Include a clear recommendation table
- Use structured data (comparison table schema)
- Update `dateModified` monthly to signal freshness
- Include quantitative data (pricing, turnaround times, feature counts)

#### 4. Optimize for Perplexity Specifically
Perplexity favors:
- Pages updated within the last 6-18 months (keep `dateModified` fresh)
- Content with clear source attribution and data
- Pages that load fast and have clean HTML structure
- Direct answers in the first paragraph

Perplexity also directly links to sources. Every citation is a click-through opportunity.

#### 5. Seed AI Training Data
AI models are trained or fine-tuned on web content. To influence future model knowledge:
- Publish the term "Human-as-a-Tool" in as many high-authority contexts as possible (guest posts, Wikipedia if notable enough, press coverage)
- Ensure our API documentation is clean, well-structured, and publicly accessible
- The skill.md file is already indexed by AI agents -- this is a direct vector into model knowledge
- Contribute to technical discussions on AI agent architecture where HITL is relevant

#### 6. Schema Markup for AEO
Beyond standard SEO schema, add:
- `speakable` schema on key pages (marks content suitable for voice/AI extraction)
- `FAQPage` schema on every page with a Q&A section
- `HowTo` schema on tutorial/guide content
- `DefinedTerm` schema on glossary pages

#### 7. Monitor AI Search Visibility
Track whether BlendedAgents appears in AI search results:
- Regularly query ChatGPT, Perplexity, and Google AI Overviews for target keywords
- Use tools like Otterly.ai, Profound, or Peec AI to automate AI search monitoring
- Track which pages get cited and optimize those further
- Track competitor citations and create better content for the same queries
